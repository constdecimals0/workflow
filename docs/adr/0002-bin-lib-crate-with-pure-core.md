# One bin+lib crate with a ratatui-free core and injected time/randomness

The example app is a single standalone cargo package (`example/app/`, `simon-says`, edition 2024, `rust-version = "1.88"`) with both a lib and a bin target — not a workspace, and not a modules-only bin. The lib is the game: `game.rs` (state machine, Sequence, scoring, Speed Tiers) and `highscore.rs` (persistence per ADR 0001), std-only with no ratatui imports. The bin is a thin shell: `main.rs` (tick loop, key→Pad mapping) and `ui.rs` (all rendering). Two seams keep the core deterministic under test: time never originates inside the lib — every time-sensitive API takes `now: Instant` as a parameter — and randomness comes from a hand-rolled xorshift64 PRNG seeded via `Game::new(seed: u64)` (the shell seeds from `SystemTime`; tests fix the seed). We deliberately rejected the `rand` crate (a dependency tree for one uniform pick of 4), a two-crate workspace (ceremony without benefit at this size), and ratatui `TestBackend` UI tests (brittle against polish tweaks, and they'd force ratatui into the lib) — don't "fix" any of these. Tests are core unit tests plus one `tests/full_run.rs` integration test that plays a scripted Run through the public API; the shell is verified by playing. The verification gate for every implement session, run from `example/app/`: `cargo test`, `cargo clippy --all-targets -- -D warnings`, `cargo fmt --check`.
